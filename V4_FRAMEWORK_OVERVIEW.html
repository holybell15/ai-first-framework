🔄 Task-Master Dispatcher

職責：分派（Route）→ 接收信號並路由（Receive Signal & Route）→ 交接（Handoff）

不做：不偵測 drift、不載入完整技術 context、不寫 code、不做設計、不做 Gate 審查

Context：必載 STATE.md + TASKS.md；drift 時讀 ARTIFACTS.md

📊 DRIFT_SIGNAL 路由

Signal Type來源Task-Master 路由
scope_driftSpecialist 發現需求超出 SRS退回 Discover → 重過 Discover Gate

test_escalation測試顯示上游文件有誤判斷是 code bug（留 Build）還是上游問題（退回）

🚪 Quality Gates — 4 個品質關卡

Gate 失敗 → 列具體未達標項（附證據）→ 判斷問題層 → 退回修正（T4 原則）→ 重過 Gate

Discover Gate Discover → Plan

  • SRS 完整：功能清單、無歧義、優先級明確
  • AC 可測試：驗收條件業務價值清晰
  • WBS 已拆至 Task 級別（S/M/L 估算）
  • MVP 邊界已確認、Stakeholder 已簽核
  • DESIGN.md 已填寫（3 方案已評估、最終方案鎖入）

Plan Gate Plan → Build

  • SD Checklist 7 項全通過：
  •   SD-1 架構圖（Mermaid/ASCII）標明所有模組 + 介面
  •   SD-2 資料流圖（Mermaid sequence）每個核心 API
  •   SD-3 DB Schema（ERD + DDL）table + relation + index
  •   SD-4 狀態機圖（有狀態變遷的核心實體）
  •   SD-5 錯誤處理策略（Error code + HTTP status + fallback）
  •   SD-6 測試矩陣（Feature × 測試層級 覆蓋規劃）
  •   SD-7 ADR 清單（所有架構決策有記錄）
  • Design Baseline locked → ARTIFACTS.md 標記 Baselined
  • Test Matrix 存在、每個 Feature 有規劃數量
  • Slice Backlog 已建立

Build Gate Build → Verify

  • 所有 Slice 實作完成
  • L1 Unit Test 覆蓋率 ≥ 80%
  • L2 Integration Test 每個 endpoint happy + error path
  • Code Review 通過（無待修評論）
  • 無 scope drift（對照 SRS）
  • API contract 符合 Plan 階段定義
  • CI pipeline 全綠

Ship Gate Verify → Ship

  • L3 E2E Test 每個 critical journey 通過
  • L4 Security 掃描完成、無 critical/high 未修復
  • L5 Smoke Test 核心路徑 100%
  • 合規審查完成（如適用）
  • Rollback plan 已驗證可 2 分鐘內完成
  • Release evidence 齊全
  • Monitoring/alerting 已配置

🏛️ 設計原則 (Design Tenets)

遇到取捨時，以原則編號小的優先。
T1
Evidence over Assertion
所有宣稱必須附帶可驗證的證據。不是 checklist 打勾，而是跑實際指令、看實際 output。
T2
Scope is Sacred
Discover Gate 通過後，需求文件是唯一範圍基準。No silent scope change / redesign / requirement rewrite。
T3
Context is Currency
每多載入 1K tokens 的 context 就是在花錢。Agent 只載入當前任務需要的最小 context。
T4
Fix Upstream, Not Downstream
若發現需求/設計有誤，回上游文件修正後再重新往下執行。不在下游硬修。
T5
Process over Guessing
遇到問題用結構化流程解決，不用直覺猜。

⚡ 關鍵規則

1. Handoff Protocol
Agent 之間不直接溝通，全部透過 Artifact + Task-Master 中轉
2. CIA 強制
修改 Baselined 文件必須走 Change Impact Assessment 流程
3. DESIGN.md 鎖定
Design Token 修改分 4 級（Free/Review/CIA/禁止）
4. Bug Fix = Regression Test
任何 bug fix 必須附帶自動化 regression test
5. XL Task 強制拆分
估算 > 4hr 的 Task 強制拆成多個 M 或 S
6. Context > 60% 存檔
自動將狀態寫入 STATE.md，確保下一個 session 可接續
