⚡ 30 秒快速開始
════════════════════════════════════════════════════════════════════════════

【步驟 1】選擇需要的 SEED 檔案
   例如：SEED_Interviewer.md（新功能討論）

【步驟 2】複製檔案內容

【步驟 3】替換所有 [佔位符]
   - [產品名稱] → 你的產品名
   - [前端框架] → Vue 3 / React / ...
   - [後端框架與語言] → Java Spring Boot / Node.js / ...
   - [資料庫] → MySQL 8.0 / PostgreSQL / ...
   - 依此類推...

【步驟 4】在 Claude.ai 新對話貼上更新後的內容

【步驟 5】開始提問或描述需求，Claude 會進入對應角色

❓ FAQ
════════════════════════════════════════════════════════════════════════════

Q: SEED 檔案可以修改嗎？
A: 可以。這是範本，根據專案需求調整沒問題。

Q: 可以跳過某些 Agent 嗎？
A: 可以。按照專案需求選擇需要的 Agent。

Q: 多個 Agent 協作時如何交接？
A: 在 TASKS.md 記錄每個 Agent 的產出與下一步，確保信息流通。

Q: SEED 可以用在多個專案嗎？
A: 完全可以。這就是通用版的設計目標。

Q: 檔案格式有規定嗎？
A: 有。見 README.md 的「文件格式標準」章節。
