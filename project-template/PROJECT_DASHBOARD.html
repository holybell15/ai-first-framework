🔀 工具切換流程（Cowork ↔ Claude Code）
AI-First 工作流分兩個階段，Gate 1 通過是唯一的切換點：
階段工具執行內容適合原因
需求階段 Cowork P01 需求訪談（Interviewer → PM → UX） 文件產出為主，不需要 Git / 編譯環境
技術階段 Claude Code P02～P06（技術設計 → 部署上線） 需要真實開發環境、Git、測試執行
Gate 1 通過後的切換步驟：
1. 打開終端機
2. cd [專案路徑] → claude
3. 進入後說：「讀取 CLAUDE.md，接續 P02 技術設計」
Claude Code 會自動讀取 Cowork 產出的所有文件（IR、US、Prototype），無需手動搬移。
Context 銜接機制：兩個工具指向同一個專案資料夾，透過 CLAUDE.md、TASKS.md、memory/ 實現完整 context 傳遞。
🔍 Gate Review 獨立 Session 操作指南
核心原則：寫文件/寫程式的 session 不做 Review。Review 必須開新 session，避免「自己改完自己審」的慣性偏差。
💬 Cowork 開新 Review Task
1. 開一個新的 Cowork task
2. 選擇同一個專案資料夾（關鍵！）
3. 第一句話（複製貼上）：

你是 Review Agent。請讀取 CLAUDE.md，然後執行 Gate [N] 驗收。

4. Review 完成後，回到原 task 繼續

1. 開一個新的終端機視窗（不要用正在開發的那個）
2. cd [專案路徑] → claude
3. 第一句話同左

4. Review 完成後，回到原終端機繼續

🤖 Agent Teams — 多 AI 並行協作（實驗性）
Agent Teams 讓多個 Claude Code AI 實例同時工作、直接溝通，無需手動切換。task-master 作為 team lead 自動派遣工作。
⚡ 啟用方式（一次性設定）
編輯 ~/.claude/settings.json，加入：

"env": {
  "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
}
需要 Claude Code v2.1.32+（目前版本已符合）
設定完成後重啟 Claude Code 生效。
🆚 Subagent vs Agent Teams
Subagent（現有）：在同一 context 內，agent 完成後回報，主流程繼續。

Agent Teams（啟用後）：多個獨立 context 同時跑，agent 之間可以直接溝通，不用過主流程。

適合：並行 Wave（Architect + DBA 同時跑）、多模組同時開發。
1
讓 task-master 當 team lead
遇到多 Feature 並行或不知從哪開始時，直接說：
我來了，要做什麼？複製
task-master 會讀 TASKS.md + STATE.md，判斷優先任務，派給正確的 Agent 或 @成員。
2
並行 Wave 範例（P02 技術設計）
啟用 Agent Teams 後，Wave 1 不用手動切換：
手動切換（啟用前）Agent Teams（啟用後）
你切到 Architect → 等完成 → 你切到 DBAtask-master 同時派出 Architect + DBA，兩個同時跑
需要人工介入每次切換Wave 1 完成後自動觸發 Wave 2

❓ 常見問題
Q：project-init.skill 在哪裡？
A：在 Template 資料夾根目錄。如果找不到，聯絡建立 Template 的人取得 .skill 檔案。
Q：Dashboard 的資料怎麼更新？
A：打開 PROJECT_DASHBOARD.html，搜尋 PROJECT_STATUS，按照 HOW TO UPDATE 說明改對應欄位，存檔後重新整理瀏覽器即可。
Q：文件命名規則是什麼？
A：F## 是功能編號（F01 = 第一個功能），類型縮寫：US（User Story）、API、TC（Test Case）、TR（Test Report）。版本從 v0.1 開始，Review 通過後升為 v1.0。
Q：Pipeline 和直接叫 Agent 有什麼差別？
A：Pipeline 自動串接多個 Agent 並在 TASKS.md 寫入交接摘要；直接叫 Agent 適合單一任務、快速修改。沒把握就用 Pipeline。
Q：owner 欄位要填什麼？
A：填執行這個 Pipeline/任務的實際負責人姓名，例如 王小明。這個欄位會顯示在 Tab 5 的時間軸和任務卡片上，方便 Team Lead 追蹤。
★ HOW TO UPDATE（每次框架有異動時更新此區塊）
1. 在 RELEASE_NOTES 陣列的最前面新增一個物件（最新版本放最上面）
2. 同步更新 VERSION 檔案和 CHANGELOG.md
3. type 值：'added'（新增）｜'changed'（修改）｜'fixed'（修復）｜'removed'（移除）
4. 存檔後重新整理瀏覽器即可看到最新版本
