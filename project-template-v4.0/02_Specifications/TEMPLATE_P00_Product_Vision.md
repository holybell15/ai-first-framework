## 📋 P00 完成度自檢

在交給 P01 Pipeline 前，確認以下項目：

```
□ 第一章完整 — 產品定義 + 適用場景 + 規模數字都填了
□ 第二章完整 — 每個功能有：概述 + 操作步驟 + 欄位清單 + 權限 + 注意事項
□ 第三章至少有 — 核心功能的驗收標準草稿（不必每個功能都有）
□ 第四章至少有 — 技術棧確定程度 + 多租戶選擇 + NFR 目標值
□ 第五章完整 — MVP / Phase 2 明確切分 + Kill Criteria
□ 功能描述的深度 — 每個功能至少有欄位清單（讓 AI 能推導 Schema）
□ 規模數字 — 座席數/並發數/資料量都有（讓 AI 能選架構）
□ 角色權限 — 至少列出主要角色的 CRUD 權限（讓 AI 能建 RBAC）
```
