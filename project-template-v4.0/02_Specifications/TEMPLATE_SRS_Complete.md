## 📋 SRS 完成度自檢（Gate 1 前必須全通過）

### 結構完整性

```
□ 封面含：產品名/文件編號/版本號/機密等級/日期
□ 版本歷程有至少一筆記錄
□ 目錄和實際章節對應
□ 第一章（系統定位）三個子節都有實質內容
□ 第二章（系統範圍）含責任劃分和資料隔離
□ 功能規格章節數 ≥ 功能模組數
□ SLA 非功能指標有具體數字（不是「良好」）
□ MVP / Phase 2 範圍明確切分
```

### 功能規格品質（抽查 3 個功能）

```
□ 7 區塊完整（功能說明/角色/流程/情境/欄位/限制/AC）
□ 操作流程每步有「使用者做 X → 系統做 Y」
□ 情境 ≥ 4 種（正常/錯誤/邊界/降級）
□ 欄位表有型別+必填+限制
□ AC ≥ 5 條，三段式（前置+操作+預期）
□ 無模糊用語（「適當」「合理」「良好」）
□ Phase 2 功能有標記
```

### AI 開發可用性

```
□ AI Agent 讀完「角色定義」能建出 RBAC 矩陣
□ AI Agent 讀完「操作流程」能設計 API endpoint
□ AI Agent 讀完「欄位規格」能設計 DB Schema
□ AI Agent 讀完「情境描述」能寫 error handling
□ AI Agent 讀完「AC 表格」能寫 Test Case
□ AI Agent 讀完「SLA」能選擇架構方案
```
