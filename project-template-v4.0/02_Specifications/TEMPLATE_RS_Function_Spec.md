## 4. 情境描述

> **寫法要求**：至少覆蓋 4 種情境。正常、錯誤、邊界、降級各至少一個。
> 品質基準：參考 AICC-II v4.0.4 的「情境 A/B/C/D/E/F」寫法。

| 情境 | 類型 | 描述 | 系統行為 |
|------|------|------|---------|
| **情境 A** | 正常 | [正常操作流程] | [成功結果] |
| **情境 B** | 錯誤 | [輸入錯誤/驗證失敗] | [錯誤提示，不洩漏內部資訊] |
| **情境 C** | 邊界 | [極端值/空值/超大量] | [系統如何處理] |
| **情境 D** | 降級 | [外部系統不可用/網路斷線] | [降級方案/提示訊息] |
| **情境 E** | 權限 | [無權限角色嘗試操作] | [拒絕方式（不顯示/403/提示）] |
| **情境 F** | 多租戶 | [A 租戶嘗試存取 B 租戶資料] | [強制隔離，回傳空結果或 403] |

## 6. 條件限制

> **寫法要求**：列出技術約束、安全要求、合規限制。

```
(a) [技術限制：如「密碼原則由租戶自行設定」]
```
