## 6. 範圍變更管理

> Gate 2 通過後，此文件即為 Baseline。任何範圍變更需遵循以下流程：

```
發現範圍需變更
  ↓
執行 /cia（Change Impact Assessment）
  ↓
評估影響：時程 / 成本 / 技術風險
  ↓
PM + 利害關係人決策：接受 / 拒絕 / 延後
  ↓
若接受 → 更新本文件 + 受影響的 US / API Spec / DB Schema
         → 版本號升版（v1.0 → v1.1）
         → 記錄變更理由到 memory/decisions.md
```
