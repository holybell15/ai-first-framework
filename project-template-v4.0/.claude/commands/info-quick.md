Quick Mode — 跳過完整 Pipeline，直接執行小範圍變更（GSD §37）。

**適用條件（三個都必須符合）：**
- 影響檔案 ≤ 3 個
- 無新功能（只是修正、調整、文字更新）
- 可在 5 分鐘內驗證完成

**Step 1 — 確認適用性**
詢問：「這個任務影響幾個檔案？有沒有新功能？」

若不符合 Quick Mode 條件，停止並說：「此任務建議走正常 Pipeline，請用 /pipeline 選擇對應階段。」

**Step 2 — 確認變更範圍**
詢問：「要改什麼？（一句話描述）」

輸出 Diff Scope Declaration（DSV-01）：
```
📋 Quick Mode 變更聲明
目標檔案：[列出檔案]
預期變更：[描述]
No-Touch List：[明確列出不可觸碰的檔案/目錄]
```

**Step 3 — 執行變更**
直接執行，不跑 Pipeline 流程。

**Step 4 — 驗證（必做）**
執行後確認：
- 變更符合 DSV-01 聲明範圍
- 若有程式碼：語法無誤、相關測試通過
- 若有文件：前後一致、無矛盾

輸出驗證結果：✅ 通過 / 🔴 發現問題 [描述]

**Step 5 — 完成**
```
⚡ Quick Mode 完成

變更：[描述]
影響：[N] 個檔案
驗證：✅

記得執行：
  git add [檔案列表]
  git commit -m "quick: [變更描述]"
```

> 若驗證失敗，自動觸發 `systematic-debugging` skill 找根因，最多 3 輪（AFL §36）。
