# Lite Task Demo

> 這是一個最小示範專案骨架，展示 AI-First Framework 在 Lite Mode 下，第一個 feature 的最小閉環應該長什麼樣子。

---

## Demo 情境

- 產品：簡易任務管理工具
- 第一個功能：F01 建立任務
- 模式：Lite Mode
- 目標：示範最少需要哪些檔案，才能讓下一個人或下一個 session 接手

---

## 你可以看到什麼

- `CLAUDE.md`：專案導航入口
- `MASTER_INDEX.md`：F-code 與正式文件登記
- `TASKS.md`：任務與交接摘要
- `memory/STATE.md`：恢復點
- `02_Specifications/US_F01_CreateTask.md`：最小需求定義
- `03_System_Design/F01-MINI-DESIGN.md`：最小設計定義
- `08_Test_Reports/F01-LITE-TR.md`：最小測試證據

---

## Demo 重點

這個示範不是完整產品，而是回答這個問題：

> 「第一次用框架時，至少要做到什麼程度，才算真的跑完第一個 feature？」

答案是：

1. 有 F-code
2. 有需求與 AC
3. 有最小設計
4. 有程式碼或明確實作範圍
5. 有測試證據
6. 有 `TASKS.md` 與 `STATE.md` 可接手
